## **1\. 상속 (inheritance)**
</br>

### **상속이란**
</br>

-   생성자와 초기화 블럭은 상속되지 않고 멤버만 상속됨
</br>
</br>

### **상속관계와 포함관계**
</br>

-   클래스 간의 관계는 딱 떨어지는 것은 아니지만, 적절한 관계를 통해 코드의 재사용성을 높일 수 있음
-   '~은 ~이다' 이면 상속 관계, '~은 ~을 가지고 있다' 이면 포함 관계

ex.

```
// 포함관계 : 원은 점을 가지고 있다. -> 포함관계를 맺는 것이 더 옳음
class Circle {
    Point c = new Point();
    int r;
}

// 상속관계 : 원은 점이다.
class Circle extends Point {
    int r;
}
```
